Apache Log4j2漏洞

漏洞简介

Apache Log4j2是一个基于Java的日志记录框架,当前被广泛应用于业务系统开发,开发者可以利用该工具䩕程序的输入输出信息进行日志记录.

在Log4j2中,日志记录功能通常使用JNDI查找来完成.JNDI查号允许应用程序访问LDAP,RMI等远程服务.如果程序未对字符合法性进行严格的限制,攻击者就可以利用这一点来注入恶意代码,控制日志消息的内容,出发远程代码执行(RCE).

漏洞适用版本: 2.0<=Apache log4j2<=2.14.1

实验环境

靶场环境: vulfocus/log4j2-rce-2021-12-09

JNDI注入工具: JNDI-Injection-Expoit-1.0-SNAPSHOT-all.jar(工具网站: https://github.com/welk1n/JNDI-Injection-Exploit/releases/tag/v1.0)

启动靶场(以我的靶场为例)

  1. 拉取log4j2漏洞的靶场镜像: docker pull vulfocus/log4j2-rce-2021-12-09
  2. 创建容器并运行: docker run -tid -p 8080:8080 vulfocus/log4j2-rce-2021-12-09
  3. 开启8080端口,访问web服务

使用DNS外带进行测试

​

​

访问成功后,查看dnslog的dns查询记录,出现payload的字段则说明测试成功

利用JNDI反弹shell

  1. 准备bash反弹命令,并将此命令进行base64编码
