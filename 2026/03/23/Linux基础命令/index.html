1-8 管道符

| 将前一个命令的标准输出（stdout） 作为后一个命令的标准输入（stdin），实现高效的数据流处理

1-9 重定向

类型 操作符 用途
重定向输入 < 从指定的文件读取数据，而不是从键盘输入
重定向输出 > 将输出结果保存到指定的文件（覆盖原有内容）
追加重定向输出 >> 将输出结果追加到指定的文件
标准错误输出 2> 将错误信息保存到指定的文件（覆盖原有内容）
追加标准错误输出 2>> 将错误信息追加到指定的文件中
混合输出 &> 将标准输出、标准错误的内容保存到同一个文件中
错误输入到标准输出 2>&1 将错误信息输出到标准信息当中

1-9-1 2>&1解释

  • 0（stdin）：标准输入（键盘）
  • 1（stdout）：标准输出（屏幕）
  • 2（stderr）：错误输出（屏幕）

command 2>&1 > log.txt，这里的 &1 指向的是屏幕而非文件，这是由 Bash 重定向的执行顺序决定的。以下是详细解析：

📌 关键原理：重定向的实时性与执行顺序

Bash 解析重定向时遵循 从左到右的即时执行原则：

  1. 当遇到 2>&1 时，系统立刻检查此时 &1（stdout）的位置
  2. 此时 > log.txt 尚未执行 → stdout 仍在默认位置（屏幕）
  3. 因此 2>&1 将 stderr 指向了当前的 stdout（屏幕）

🧩 为何 &1 在此场景指向屏幕？

  • &1 是一个 动态指针，始终指向 stdout 当前的位置

  • 在

    被解析的瞬间：

    • > log.txt 还未执行 → stdout 仍在屏幕
    • 因此 &1 指向屏幕

  • 核心规律：&1 的指向取决于执行到该符号时 stdout 的状态

⚠️ 图片注释的深层含义

图片中 # 先"跟风"再"定目标" 的注释正是警示：

  1. 先 “跟风”（2>&1）
    让 stderr 盲目跟随了此刻的 stdout（屏幕）​​
  2. 后 “定目标”（> log.txt）
    仅改变了 stdout 的目标，​未改变 stderr 的指向​

💎 总结

在命令 command 2>&1 > log.txt 中：

  1. &1 指向屏幕，因为执行到 2>&1 时，stdout 尚未被重定向到文件
    （> log.txt 在右侧尚未生效）
  2. 错误输出会显示在屏幕，只有正常输出写入文件
    ​根本原因​：Bash 的重定向是 ​即时动态生效​ 的，顺序决定一切

记忆口诀：&1 像实时导航，永远指向 stdout 此刻的位置！

1-10 &&逻辑与符号

在Linux中，&& 符号是一个逻辑与运算符，用于连接两个或多个命令。其具体作用是：当且仅当 && 左边的命令成功执行（即返回状态码为0）时，才会执行 && 右边的命令
